<script type=”text/javascript”>

// 函数

/*

function 函数名([参数1，参数2, …]){

//函数体

}

函数定义后不会被调用，只有显式的调用才会执行函数里面的代码

*/

// 不带参数的函数

function test1(){

document.write(‘test1’);

}

// 函数的调用

test1();

// 带一个参数的函数

function test2(msg){ // msg:形参 (形式参数)

document.write(msg);

}

// 调用带参数的函数，传入参数

test2(‘ hello’); // 传入的参数 ‘hello’ （实参）

// 定义一个带参数的函数，可不可以不传参数去调用这个函数呢？

function test3(arg){

document.write(12312);

}

test3();

function test4(arg){

if(arg){

document.write(arg);

}

}

test4();

test4(‘<br/> test4’);

// 把累加封装成一个函数：必须能通过参数调节累加量，如果不设置参数那么就累加到100

function add(num){

/*

if(!num){

num = 100;

}*/

num = num ? num : 100;

for(var i=1,sum=0;i<=num;i++){

sum += i;

}

alert(sum);

}

add(1000);

add();

</script>

刚刚学到js的类型，发现js的和PHP的有很大的区别.主要是在逻辑值 true和false上。我们来个例子看下：
<script>

</script>

-2是非0的数，按理说应该输出应该是true,可是这里输出却是false，为啥呢..因为js语法的独特性，它会把true变成”1”再和”-2”做比较，结果就肯定是false了

<?php
$a=true==’-2’;
echo $a;

css hack技术

发表于 2012-04-29   |   分类于 学习笔记 
很多时候我们写出来的CSS样式，在每种浏览器出现的效果可能有很大的差别，比如火狐，默认的是16PX的字体,不过这些可以通过对浏览器初始化来实现.   今天来介绍下[**CSS hack** ](http://www.dudu.me "css back")技术，以某一个浏览器专有的方式来写，让我们需要的浏览器读懂我们的样式。

background:#F00;/所有浏览器都会成红色/
background:#00F\9;/只有IE系列的浏览器才会变成蓝色/
background:#0F0;/只有IE6 7才会执行/
_background:#CFF;/只有IE6才会执行*/

这是常见的几种浏览器的特有的，假如我们要在IE7 中，实现边框的颜色为黑色，而在其他的IE6 8 火狐下都显示红色边框我们又怎么来做呢？

只有\9 ，，_这三种浏览器的hack，当因为兼容性问题，而需要使用H*ACK技术的时候，那么，必须先从范围广的写法开始写，然后逐渐写上控制范围较小的写法，来逐步控制到你要控制的浏览器

使用类似办法可以得到IE7的外边框为黑色，而其他浏览器为#ff0000。

border:10px solid #ff0000;
*border:5px solid #000;
_border:5px solid #ff0000;
